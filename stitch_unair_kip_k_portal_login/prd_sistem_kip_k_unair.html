# PRD: Sistem Pendaftaran KIP-K Universitas Airlangga (UNAIR) ## 1. Autentikasi - **Login (#login)**: Form email/password, tombol masuk dengan loading, alert error, link register. Role-based redirect (Admin/Mahasiswa). - **Register (#register)**: Nama, email, password (min 8 char). Auto-login setelah sukses. ## 2. Role Mahasiswa - **Dashboard (#mhs)**: Tombol "+ Ajukan KIP-K", tabel riwayat pengajuan (ID, Status, PDF, Tanggal). Empty state jika belum ada data. - **Form Pengajuan (#mhs-form)**: 5 Section (Dokumen, Penghasilan, Kondisi Keluarga, Standar Hidup, Upload PDF Ditmawa). Validasi wajib PDF. - **Detail Detail (#mhs-detail)**: Status badge besar, checklist PDF, log riwayat status, catatan admin jika ditolak. ## 3. Role Admin - **Dashboard (#admin)**: 6 Stats cards, tabel pengajuan dengan filter status & prioritas. - **Detail Pengajuan (#admin-detail)**: Data raw (encoded), link PDF, perbandingan hasil AI (CatBoost vs Naive Bayes), aksi verifikasi/tolak/koreksi. - **Koreksi Data (#admin-correction)**: Grid dropdown untuk koreksi 13 field encoding untuk data training. - **Retrain Model (#admin-retrain)**: Penjelasan model, tombol mulai retrain, ringkasan akurasi setelah proses. ## 4. Teknis & Routing - SPA (Single Page Application) dengan `goPage(id)`. - Auth Guard & Role-based access. - API endpoints terdefinisi untuk setiap aksi. - Helper: format Rupiah, format tanggal, loading state, alert handler.
